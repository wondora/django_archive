Dockerfile이나 entrypoint를 건드리기 귀찮다면, 코드를 수정하거나 배포할 때마다 수동으로 딱 한 번 명령어를 쳐주면 됩니다.
docker-compose exec web python manage.py collectstatic --noinput
이 명령을 해야 관리자 페이지가 안깨진다.

--------------
다른 복잡한 검사(숫자만 쓰면 안 됨, 너무 쉬운 비밀번호 안 됨 등)를 모두 제거하고, 오직 "4자리 이상인가?" 만 검사하도록 설정한 코드입니다.

🛠️ settings.py 수정 코드
Python
# 비밀번호 유효성 검사 설정
# (기존 내용은 다 지우고 이 코드로 교체하세요)

AUTH_PASSWORD_VALIDATORS = [
    {
        'NAME': 'django.contrib.auth.password_validation.MinimumLengthValidator',
        'OPTIONS': {
            'min_length': 4,  # 최소 4자리부터 허용
        }
    },
]
✅ 이렇게 바꾸면?
비밀번호를 "1234" 처럼 숫자 4자리로만 만들어도 통과됩니다. (기존에는 숫자로만 만들면 에러가 났습니다.)

"password" 같은 쉬운 단어도 사용 가능해집니다.

관리자 페이지(/admin/)나 회원가입 로직에서 4글자 이상이면 무조건 통과됩니다.

지금 작성해 드린 코드는 **"기본적으로는 모든 권한(업로드/삭제)을 주고, 'ReadOnly' 그룹에 속한 사람만 권한을 뺏는 방식"**으로 작동합니다.
따라서 권한 관리는 아주 심플합니다.
1. 읽기 전용 (다운로드만 가능) 계정 만들기관리자 페이지(/admin/)에서 다음과 같이 설정하면 됩니다.
Groups(그룹) 메뉴에서 ReadOnly 라는 이름의 그룹을 만듭니다. 
(대소문자 정확히)Users(사용자) 메뉴에서 계정을 클릭합니다.
Groups 항목에서 ReadOnly를 선택해서 오른쪽으로 넘깁니다.
[중요] Superuser status(최고 관리자 권한) 체크박스는 반드시 해제해야 합니다. 
(이게 체크되어 있으면 그룹 상관없이 무조건 다 됩니다.)저장합니다.
👉 결과: 이 사람은 파일 목록 보기와 다운로드만 가능하고, 업로드/삭제 버튼이 사라집니다.
2. 업로드/다운로드 가능한 (일반) 계정 만들기특별히 무언가 설정을 할 필요가 없습니다.
Users(사용자) 메뉴에서 계정을 만듭니다.Groups 항목을 비워둡니다. (아무 그룹에도 넣지 마세요.)저장합니다.
👉 결과: 'ReadOnly' 그룹이 아니므로, 자동으로 업로드/삭제/폴더생성 등 모든 기능이 활성화됩니다.
